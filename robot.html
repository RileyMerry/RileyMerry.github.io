Network-Controlled Ground Robot

A remotely operated ground robot designed to demonstrate embedded control, client/server architecture, and hardware/software integration.

Project Overview

This project uses a Raspberry Pi as a Linux-based control computer for a small unmanned ground vehicle. A browser-based interface sends movement commands over WiFi to a C++ server running on the Pi, which then controls GPIO pins connected to a motor driver.

System Architecture

Browser → WiFi → Raspberry Pi → C++ Server → GPIO → Motor Driver → Motors

Software

  • C++ backend server for command handling
  • Browser-based control interface
  • GPIO motor control abstraction
  • CMake-based build workflow
  • Linux/systemd service setup

Hardware

  • Raspberry Pi control computer
  • Motor driver interface
  • External motor battery supply
  • Shared ground between logic and motor power
  • Custom chassis development

Project Media

Engineering Challenges

Key challenges included debugging power delivery, understanding shared grounding between the Pi and motor driver, configuring Linux-based development tools, and designing software that separates the web interface from the hardware control layer.

Future Improvements

  • Live camera streaming in the web interface
  • Wheel encoders for feedback
  • PID speed control
  • Obstacle detection sensors
  • Semi-autonomous driving modes
