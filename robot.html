UGV Drone Project

Unmanned ground vehicle built around embedded control, networking, and system design.

This project combines Raspberry Pi hardware, C++ software, GPIO motor control, Linux services, browser-based operation, CAD design, and TCP/IP networking into a functional unmanned ground vehicle platform.

View Case Study GitHub Repository

Target User & Mission Vision

The envisioned target users for this platform are military, law enforcement, and emergency-response teams that may need a small ground vehicle for inspection, scouting, training, or remote observation in areas where sending a person first may be unnecessary or unsafe.

The purpose of this project is not only to build a moving robot, but to explore how embedded control, communication, mechanical design, and software architecture come together in a real system.

CAD Model & Physical Design

The first chassis design focuses on getting the system functional before optimizing for weight, material usage, sensor mounting, and user testing. The goal is to create a simple but reliable test platform that can evolve into a more refined final design.

Future design improvements may include improved motor mounting, cleaner cable routing, camera placement, sensor integration, and a lighter chassis optimized through testing.

Networking Demonstration

The robot is controlled through a browser-based interface where the Raspberry Pi acts as the server and the browser acts as the client. Communication occurs over TCP using a local network connection.

The Wireshark capture below shows the client initiating a connection to the Raspberry Pi server on port 8080. This validates that the robot control interface is communicating over a real client-server network connection.

This capture shows the initial SYN packet used to establish a TCP connection between the client and the Raspberry Pi server. This is evidence of transport-layer communication between the web interface and the robot control system.

Code Architecture

The software is organized around C++ source files, a CMake build structure, Linux scripts, GPIO control, front-end browser interaction, and backend logic that translates user input into motor control behavior.

Software Stack

The project uses a modular layout so the embedded control logic, build system, scripts, and interface code can be developed and tested independently.
